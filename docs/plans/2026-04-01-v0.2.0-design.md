# hs-net v0.2.0 设计方案

## 目标

降低使用门槛，让用户更愿意尝试和使用 hs-net。

## 变更内容

### 1. 可选依赖拆分

**核心依赖**（`pip install hs-net`）：
- httpx[http2,socks] — 默认引擎
- pydantic, blinker, tenacity, furl, fake-useragent, parsel, jmespath

**可选依赖组**：
- `hs-net[aiohttp]` → aiohttp
- `hs-net[curl]` → curl-cffi
- `hs-net[requests]` → requests
- `hs-net[requests-go]` → requests-go
- `hs-net[all]` → 全部引擎

**引擎懒导入机制**：
- 各引擎模块顶部尝试 import 对应库，失败时设置标志位
- 引擎类实例化时检测标志位，缺失则抛出 `EngineNotInstalled` 异常
- 异常消息包含安装命令：`pip install hs-net[xxx]`

### 2. 函数式 API

**模块级快捷函数**：
- 异步：`hs_net.get()`, `hs_net.post()`, `hs_net.put()`, `hs_net.patch()`, `hs_net.delete()`, `hs_net.head()`, `hs_net.options()`, `hs_net.request()`
- 同步：`hs_net.sync_get()`, `hs_net.sync_post()` 等同名函数
- 每次调用创建临时客户端，执行请求后自动关闭
- 参数签名与 Net/SyncNet 方法一致，额外支持 engine 参数

### 3. 引擎能力矩阵文档

README 新增引擎对比表，覆盖：
- 异步/同步支持
- HTTP/2 支持
- TLS 指纹模拟
- 代理类型（HTTP/SOCKS）
- 推荐使用场景

## 决策记录

- 只保留 httpx 作为核心依赖，其他引擎全部可选
- 函数式 API 采用每次创建临时客户端的方式，不用单例
- 引擎缺失时延迟到使用时报错，不在 import 时 warning
